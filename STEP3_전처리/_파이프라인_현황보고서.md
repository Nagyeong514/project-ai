# 📋 전처리 파이프라인 현황 보고서

> 각 모델의 역할 · 넣은 프롬프트가 어디까지인지 · 앞으로 할 일. (2026-06-30)

---

## 🗺️ 한눈에: 파이프라인 흐름과 역할 분담

```
[영상] ─┬─→ YOLO ───────→ 부품 "위치"만 (어디에 RAM/슬롯이 있나)
        └─→ VLM(Qwen3-VL) → 행동 "관찰"만 (손이 무엇을 하나)   ★프롬프트 있음

[오디오] → STT(Whisper) → 발화 전사 (말한 내용)               ★프롬프트 없음(순수 전사)
                ↓
         Transcript 정제 → 용어교정 + 반복환각 태깅            ★프롬프트 아님(규칙기반)
                ↓
   ┌──────────────────────────────────────┐
   │  LLM 융합(Qwen2.5-7B)                  │ ★프롬프트 있음(제일 두꺼움)
   │  행동 + 발화를 타임스탬프로 묶어         │
   │  "암묵지 후보" JSON 생성                │
   └──────────────────────────────────────┘
```

**핵심 설계 사상 = 역할 분리.** VLM은 "눈"(보이는 것만), LLM은 "머리"(해석·판단 전부).
그래서 프롬프트가 들어간 건 **VLM과 LLM 딱 둘**이다.

---

## 1️⃣ STT (Whisper) — 프롬프트 ❌ 없음

순수 음성→텍스트라 프롬프트 불필요. 대신 **설정으로 통제**:
- `condition_on_previous_text=False` → 반복 환각 루프 억제
- `language=ko`, `compute_type=float16`(Turing sm75)
- 모델: `faster-whisper large-v3-turbo`

**상태: ✅ CLIP1~4 전사 완료.** 여기는 "프롬프트 넣을 데"가 아님.

---

## 2️⃣ Transcript 정제 — 프롬프트 ❌ (규칙기반, (b)안)

LLM이 아니라 **결정적 코드**라 프롬프트가 아님. 딱 2가지만:
1. **용어 정규화** — 현재 활성은 **단 3개**: `마더보이드/마더보드/메인보드 → motherboard`
2. **연속반복 태깅** — 같은 문장 반복되면 `repeat_hallucination=true` (삭제는 안 함)

> ⚠️ 헷갈리기 쉬운 지점: 원래 "정규식으로 근거(인과/조건/주의) 추출"을 시도했다가
> → **한국어 구어체에서 0건 잡혀서 폐기**함. 그래서 근거 판단은 전부 LLM으로 넘겼다.
> 지금 정제는 "용어교정 + 반복태깅"만 하는 가벼운 버전(=(b)안).

**상태: ✅ 구현·검증. `transcripts/*.refined.json` 산출됨.**

---

## 3️⃣ VLM (Qwen3-VL-8B) — 프롬프트 ✅ 완성됨 (버전 A: 관찰만)

`prompts/vlm_observation.py`에 들어가 있음. 어디까지 넣었냐면 — **5대 기법 전부**:

| 넣은 것 | 내용 |
|---|---|
| ① 시스템 프롬프트 | "너는 눈이다. 보이는 사실만 적어라" |
| ② 추상동사 금지 | "확인/점검/검사/진단" 쓰지 마 → 구체 손동작으로 |
| ③ 손 단위 분리 | "어느 손/어느 손가락이 무엇을" |
| ④ 수치 추정 금지 | 각도·거리 지어내기 금지 (단 LED **횟수**는 허용) |
| ⑤ LED는 횟수만 | "황색 1회+백색 3회"까지만, **의미 해석 금지** |
| few-shot 3개 | 출력 JSON 형식 강제 |
| 메타정보 은폐 | "RAM 탈거 시뮬레이션"이란 사실 VLM에 안 줌 |

**즉 VLM 프롬프트는 다 넣었음.** 더 넣을 거 없음(채점 후 튜닝만 남음).

**상태: ✅ 프롬프트 완성. 🔲 아직 실제 영상으로 풀런 안 해봄.**

---

## 4️⃣ LLM 융합 (Qwen2.5-7B-AWQ) — 프롬프트 ✅ 완성됨 (제일 두꺼움)

`prompts/llm_fusion_prompt.py`. 파이프라인의 머리라 프롬프트가 제일 큼. 넣은 것:

| 넣은 것 | 내용 |
|---|---|
| 역할 정의 | "후보를 빠짐없이·정직하게 구조화. 진짜 암묵지 판별은 안 함" |
| **근거성 판단** | "~하면 됩니다 / 딸깍 소리 나면 잠긴 것" 같은 구어체 노하우를 네가 읽어내라 (정제에서 폐기한 걸 여기로 이관) |
| 3가지 케이스 | A(행동+발화) / **B(행동만=action_only, 누락금지)** / C(발화만) |
| 할루시네이션 금지 | 입력에 없는 사실 생성 금지, 묶기/라벨링만 허용 |
| 정직 태깅 | 추론은 `reasoning_origin="model_inferred"`, 발화인 척 위장 금지 |
| **LED 진단표** | 황1백3=메모리/프로세서 등 — **LLM만 봄(VLM엔 없음)** |
| 반복환각 무시 | `repeat_hallucination=true` 발화는 무시하라 |
| 출력 스키마 | schema_version 1.3 JSON 형태 |

**즉 LLM 융합 프롬프트도 본문은 다 넣었음.**

**상태: ✅ 프롬프트 완성. 🔲 아직 풀런 안 해봄.**

## 📊 요약: 프롬프트 "어디까지" 넣었나

| 모델 | 프롬프트 필요? | 현재 상태 |
|---|---|---|
| STT | ❌ 불필요 | ✅ 완료 |
| Transcript 정제 | ❌ (규칙기반) | ✅ 완료 |
| **VLM** | ✅ 필요 | ✅ **본문 다 넣음** |
| **LLM 융합** | ✅ 필요 | ✅ **본문 다 넣음** |

**결론: 프롬프트는 둘 다 끝까지 넣었다.** 빠진 건 프롬프트가 아니라 아래 "기능".

## ⚠️ 아직 안 끝난 것 (프롬프트 말고 "기능")

1. **스키마 `conflict` 필드** — 본 것 vs 들은 것 충돌 보존용. 미반영
2. **YOLO 백업그물** — 손-부품 접촉 이벤트 저장 + 융합 입력에 주입. 미반영
3. **풀런 자체** — CLIP1 끝까지(STT→VLM→LLM) 한 번도 안 돌려봄 ← 이게 제일 중요
