## 3. 진행 중이던 작업 (이어서 할 것)
- **CLIP2 conflict 테스트 결과: 무의미(inconclusive).** VLM이 CLIP2에서 **빈 관찰(`{"observations": []}`)** 반환 → LED 관찰이 없어 STT "백색5 황색4"와 충돌할 상대가 없음 → conflict=false 불가피.
  - 즉 **conflict 자연 발현 테스트는 VLM 품질 고친 뒤(서버)에야 가능.** 한 클립에 VLM LED관찰 + STT LED발화가 동시에 있어야 하는데, 현재 VLM이 (CLIP3=LED관찰O/STT LED발화X, CLIP2=STT발화O/VLM관찰X)로 엇갈림.
  - conflict 배관 자체는 검증됨(필드·기본값·cross_check 경고 OK, CLIP3서 false 출력 확인).
- **파서 추가 수정**: VLM이 빈 배열 반환 시 정크 1건 만들던 버그 → 깔끔히 0건 반환하도록 고침(`vlm_qwen._parse_observations`).
