# VLM 모델비교 — 질문 답변 (정답지 ↔ 모델 출력 비교)

> 대상: `51624.mp4` (델 워크스테이션 RAM·그래픽카드 조립, 164초) / Qwen3-VL-8B vs InternVL2.5-8B
> 일시: 2026-06-29 (정답지 recall 채점 추가)

## Q1. JSON 정답지와 각 모델 출력을 비교했다는 뜻인가?

**예.** 두 단계로 비교했습니다.

1. **정성 비교(육안)**: 동일 입력(같은 영상·구간·전사·프레임·프롬프트)으로 두 모델에 암묵지 후보를 뽑게 한 뒤, 같은 구간 출력을 나란히 놓고 깊이·환각 여부를 눈으로 대조.
2. **정량 비교(정답지 recall)**: 전사에서 **필수개념 9개**를 뽑은 **JSON 정답지**(`51624_answerkey.json`)를 만들고, 각 모델 출력에 그 개념(동의어 묶음 중 하나라도)이 등장하는지 자동 채점 → **recall**.

즉 "정답지의 핵심 개념을 모델 출력이 얼마나 담았나"를 대조한 것입니다. **글자 일치가 아니라 개념(동의어) 적중** 방식입니다.

## Q2. JSON 정답지 & 모델별 출력 예시

### ① JSON 정답지 (필수개념 단위)

```json
{
  "seg_idx": 0,
  "task": "RAM(메모리) 장착",
  "knowledge": "CPU1 슬롯은 비활성이라 CPU0에만 장착한다. 같은 브랜드 권장하나 호환되면 혼용 가능. 닫힌 슬롯은 먼저 열고, 접지가 중요하므로 핀 방향을 맞춰 넣은 뒤 슬롯을 닫는다.",
  "keywords": [
    ["CPU1", "액티베이트", "비활성", "논 값"],
    ["혼용", "다른 브랜드", "호환되면", "하이닉스"],
    ["먼저 열", "열어주", "열고"],
    ["접지"],
    ["핀 방향", "앞뒤", "방향 맞"],
    ["슬롯을 닫", "닫아주", "닫기"]
  ]
}
```
→ 안쪽 리스트 하나 = **필수개념 1개**. 그 동의어 중 하나라도 모델 출력에 있으면 "적중".

### ② 모델별 출력 (같은 RAM 장착 구간, 실제 출력)

**Qwen3-VL-8B** — 근거 발화까지 정확히 인용:
```json
{ "action": "RAM을 장착할 때 핀 방향을 맞춘다",
  "tacit": "핀 방향을 맞추지 않으면 장착 불가능하거나 손상될 수 있으므로 반드시 확인한다",
  "evidence": "발화 전사: '핀 방향 맞춰서 잘 넣어주시면 되겠습니다'" }
```

**InternVL2.5-8B** — 문장 깨짐·표면적:
```json
{ "action": "컴퓨터 컴파운트 열어서 램을 넣어보다",
  "tacit": "램이 닫혀 있어 열어주고 닫혀있는 상태로 넣어야 하다",
  "evidence": "CPU 슬롯이 닫혀 있고, 닫혀있는 상태로 넣어야 하며 접지가 중요하다" }
```
> InternVL은 그래픽카드 구간에선 프롬프트의 요리 예시("칼날을 15도 눕혀야…")를 그대로 베끼는 **환각**까지 발생.
