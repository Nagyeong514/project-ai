# 배치 추론 추가 실험 결과 보고서

> 작성일: 2026-06-29
> 대상: `vad_stt_research` 비배치 3-arm 실험에 배치 추론 조건 2개 추가
> 데이터: 10파일(high_silence 5 + low_silence 5), STT = faster-whisper large-v3-turbo (int8_float16), RTX 2080 8GB
> 결과 원본: `batch_addon/results_batch.csv` (40행 = 10파일 × 4조건)

---

## 1. 실험 개요

기존 비배치 결론은 "turbo 환경에서 VAD는 순손해, `A′`(파라미터만)가 최선"이었다.
용도가 **녹음 파일 일괄 처리 + 높은 처리량**이라, 배치 추론(BatchedInferencePipeline)을
켜면 결과가 달라지는지 검증하기 위해 조건 2개를 추가했다.

| 조건 | 자르는 기준 | 추론 | 비교 축 |
|---|---|---|---|
| `A_prime` | 안 자름(전체 1회) | 순차 (기존) | — |
| `A_prime_batch` | 고정 30초 윈도우(VAD 없음, 100% 커버) | 배치 | A′ ↔ A′_batch = 배치 효과 |
| `B` | 외부 Silero VAD 청크 | 순차 (기존) | — |
| `B_batch` | B와 **동일한** Silero VAD 청크를 clip_timestamps 주입 | 배치 | B ↔ B_batch = 배치 효과 |

통제·설계 상세는 `README.md` 참조. RTF 공정 비교를 위해 4조건을 **같은 세션**에서 새로 측정했다
(기존 `results_10.csv`의 옛 숫자와 직접 비교하지 않음).

**측정 지표**: WER / CER / 할루시네이션per시간 / RTF (warmup 1 + 측정 3회).
`timestamp_drift`는 배치의 `without_timestamps=True` 차이로 비교 불가하여 제외(사유 README).

## 2. 전체 평균 (10파일)

| 조건 | WER | CER | 할루시/h | RTF |
|---|---|---|---|---|
| A′ (비배치, VAD없음) | 0.224 | 0.092 | 99.8 | 0.0208 |
| **A′_batch** | 0.287 | 0.171 | **20.3** | **0.0081** |
| B (비배치, VAD) | 0.266 | 0.133 | 119.4 | 0.0383 |
| **B_batch** | 0.268 | 0.143 | 81.1 | 0.0153 |

### 배치 효과 (음수 = 배치가 개선)

| 지표 | A′ → A′_batch | B → B_batch |
|---|---|---|
| **RTF(속도)** | **−0.0127 (2.58배 빠름)** | **−0.0230 (2.50배 빠름)** |
| 할루시/h | **−79.5** | **−38.3** |
| WER | +0.063 (악화) | +0.002 (동일 수준) |
| CER | +0.079 (악화) | +0.010 (거의 동일) |

**자원**: OOM 폴백 0건, batch_size=8 전 구간 유지 (8GB에서 안정).

## 3. 무음 그룹별

| 그룹 | 조건 | WER | CER | 할루시/h | RTF |
|---|---|---|---|---|---|
| **high_silence** (간헐 발화, 5) | B | 0.178 | 0.080 | 142.2 | 0.0356 |
| | **B_batch** | **0.174** | **0.080** | 120.3 | 0.0159 |
| | A′ | 0.165 | 0.072 | 104.4 | 0.0167 |
| | A′_batch | 0.192 | 0.108 | 20.3 | 0.0076 |
| **low_silence** (연속 발화, 5) | B | 0.353 | 0.186 | 96.6 | 0.0411 |
| | B_batch | 0.362 | 0.207 | 41.8 | 0.0147 |
| | A′ | 0.283 | 0.113 | 95.1 | 0.0249 |
| | A′_batch | 0.381 | 0.234 | 20.3 | 0.0086 |

- **high_silence에선 B_batch가 사실상 공짜 이득**: WER·CER가 B와 동일(WER은 미세 개선)인데 약 2.2배 빠르고 할루시도 감소.
- **low_silence에선 배치의 정확도 손해가 큼**(CER B 0.186 → B_batch 0.207): 발화가 빽빽해 청크가 길고 많아 배치 분할 손실이 누적.
- **A′_batch의 할루시/h는 그룹 무관 20.3으로 일정**: 고정 30초 윈도우가 무음 분포와 무관하게 결정적으로 동작하기 때문.

## 4. L01 (비배치 B 과분할 케이스) 추적

| 조건 | WER | CER | n_chunks |
|---|---|---|---|
| B | 0.541 | 0.429 | 375 |
| B_batch | 0.548 | 0.438 | 375 |

배치는 과분할을 고치지 못한다(B_batch는 B와 동일한 VAD 청크 375개를 그대로 사용).
배치는 속도·할루시만 바꾸고 **청크 경계(정확도 손실 원인)는 건드리지 않는다** —
"배치 여부만" 분리하려던 설계가 의도대로 작동했다는 방증.

## 5. 결론 (녹음 파일 일괄 처리 관점)

1. **처리량이 목적이면 배치는 강력**: 2.5배 속도 + 할루시네이션 감소, 8GB에서 OOM 없음.
2. **A′_batch**(VAD 없는 빠른 처리): 가장 빠르고(RTF 0.008 ≈ 125배속) 할루시 최소지만
   **CER이 약 2배(0.092→0.171) 악화** — 고정 30초 칼분할이 문장을 끊어 글자 오류 증가. 정확도 중요 시 부적합.
3. **B_batch**: 정확도를 B 수준으로 유지하면서 2.5배 속도. **특히 high_silence(무음 많은) 콘텐츠에 최적**
   — 정확도 그대로 + 2.2배 속도 + 할루시 감소.
4. **순수 정확도 1위는 여전히 A′(비배치)** — 기존 결론 유지. 배치는 정확도를 조금 내주고 속도·할루시를 사는 트레이드오프.

### 용도별 권고

| 우선순위 | 권고 조건 | 근거 |
|---|---|---|
| 정확도 최우선 | A′ (비배치) | WER 0.224 / CER 0.092 최저 |
| 처리량 + 정확도 균형 | **B_batch** | 정확도 B 수준 유지, 2.5배 속도, high_silence에 특히 유리 |
| 최대 처리량 (정확도 양보 가능) | A′_batch | RTF 0.008(125배속), 할루시 최소, 단 CER 손해 |
