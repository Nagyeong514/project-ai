# VAD STT 연구 — Phase 1 실험 결과 보고서

> 실험일: 2026-06-24~25 | 상태: 완료

---

## 핵심 요약

| 발견 | 결론 |
|------|------|
| **파라미터 통일(A')** | RTF 47~52% 감소, WER 대폭 개선 — **효과 명확** |
| **VAD(B) 속도** | VAD 오버헤드(RTF +0.043~0.048)가 무음 제거 이득을 상쇄 → **A보다도 느림** |
| **VAD(B) 정확도** | high_silence 일부에서 WER 소폭 개선, low_silence에서 악화 — **불안정** |
| **이상치** | H02(n_chunks=0), L02(n_chunks=4) — Silero VAD 작동 실패 |

**실용 결론:** 현재 환경에서 VAD 없이 파라미터만 통일한 **A'가 가장 빠르고 안정적**이다.

## 실험 환경

- STT: faster-whisper large-v3-turbo (CTranslate2 int8_float16)
- GPU: RTX 2080 8GB
- VAD: Silero VAD (CPU, threshold=0.5, speech_pad_ms=400)
- 데이터: YouTube 수동 자막 영상 10개 (high_silence 5개 + low_silence 5개)
- 반복: warmup 1회 + 측정 3회, RTF는 평균±SD

### 1.2 핵심 발견: VAD 오버헤드가 이득을 상쇄

Silero VAD는 CPU에서 실행되며 **오디오 길이의 약 4.3~4.8%를 VAD 실행에 소모** (RTF ≈ 0.043).
large-v3-turbo가 이미 빠르기 때문에 (A' RTF ≈ 0.013~0.033) VAD로 무음을 제거해도 이 오버헤드를 회수하지 못한다.

```
A'  RTF 평균 (이상치 제외): 0.0202
B   RTF_eff 평균 (이상치 제외): 0.0773
→ VAD 적용 시 A' 대비 3.8배 느림
→ 심지어 vanilla A(0.0330) 대비도 2.3배 느림
```

**원인:** Silero VAD가 GPU가 아닌 CPU에서 실행되어 병렬화 이점이 없음. GPU 기반 VAD라면 결과가 달라질 수 있음.

## 4. 이상치 분석

### H02 — n_chunks=0 (VAD 완전 실패)

- 89분 다큐멘터리, 무음 76.5%
- Silero VAD(threshold=0.5)가 발화 구간 0개 검출
- 가능 원인: 다큐 내레이션의 낮은 음량 또는 배경음악이 VAD를 혼란시킴
- threshold를 낮추거나(0.3) 최소 speech 감지 후처리 필요

### L02 — n_chunks=4 (VAD 부분 실패)

- 97분, 무음 0.7%인데 4개 청크만 검출
- 발화가 거의 전구간인데 4개 세그먼트로만 분할 → 처리 시간 ≈ 0
- 가능 원인: 오디오 인코딩 문제 또는 채널/샘플링 포맷 불일치

### H03, H04, L04 — 전 조건 WER 80%+

- 조건과 무관하게 모든 WER이 높음 → STT 오류가 아닌 **GT 품질 문제**
- 다큐멘터리 자막이 실제 발화 텍스트와 표현 방식이 달라 WER이 부풀려짐
- 이 3개 파일은 정확도 비교에서 제외하고 해석해야 함

## 5. 종합 결론

### 가설 검증

| 가설 | 결과 |
|------|------|
| 가설 1 (VAD → 할루시네이션 감소) | **부분 지지.** A'만으로도 대폭 감소. VAD 추가 효과는 불안정 |
| 가설 2 (high_silence에서 VAD → RTF 감소) | **기각.** VAD 오버헤드(RTF+0.044)가 이득을 상쇄. B가 A보다도 느림 |
| 가설 3 (padding 부족 → WER 악화) | 민감도 분석 미실시. 향후 과제 |

### 실무 권고

현재 RTX 2080 환경(Silero VAD CPU 실행)에서는:

1. **A' 채택 권장** — 파라미터 통일만으로 RTF 50% 감소, WER 대폭 개선, 안정적.
2. **VAD(B) 보류** — 속도 이득 없음, 정확도 불안정, 일부 콘텐츠에서 완전 실패.
3. **VAD GPU 실행 검토** — Silero를 GPU로 돌리면 오버헤드(RTF 0.044) 해결 가능성 있음.

### 예상 외 발견

**large-v3-turbo가 이미 너무 빠르다.** A'의 RTF가 0.013~0.033으로, 무음을 제거해서 아낄 수 있는 시간보다 VAD 실행 비용이 더 크다. 이는 WhisperX가 검증한 large-v2 기준(RTF ~0.3 수준)과는 전혀 다른 환경이다. **모델이 빠를수록 VAD의 속도 이점은 줄어든다.**
