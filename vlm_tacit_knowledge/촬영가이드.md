# 촬영 가이드 — 연구④ 실험용 영상 (1장)

목표: **발화/침묵이 섞이고, 노하우가 발화에만 몰리지 않은** 깨끗한 실험 영상 만들기.
(이게 있어야 C 조건 켜고 B vs C 가설검증 가능)

## ⭐ 제일 중요한 원칙 — 4종류가 다 섞이게
일부러 "중요할 때만 말하기" **금지**(= 답을 미리 심는 것). 아래 4칸이 자연스럽게 다 나오게:

|  | 말함 | 조용함 |
|---|---|---|
| **중요(노하우O)** | ✅ 설명하며 시범 | ✅ **말없이 시범** ← 꼭 넣기 |
| **안 중요** | ✅ 잡담/딴소리 | ✅ 그냥 작업 |

- 특히 **"중요한데 말 안 하고 보여주는"** 구간을 의도적으로 몇 개 넣어. (발화 밖에도 노하우 있나 검증용)
- 발화:침묵 ≈ **5:5 정도** 목표 (지금 영상은 9:1이라 실패).

## 촬영 방법
- **소리 켰다 껐다** 자연스럽게 — 말할 땐 말하고, 침묵 땐 진짜 조용히(설명 X, 작업은 계속).
- **후시녹음·자막 더빙 금지** — 영상과 소리 시점이 어긋나면 실험 무효. 카메라 마이크로 현장 녹음.
- 작업 한 종류(예: 타이어 교체)당 **한 번에 쭉** 촬영.

## 화면/품질
- **손과 작업 부위가 잘 보이게** — 노하우는 디테일(각도·부품 상태)에 있음. 가까이, 밝게.
- 해상도 **1080p 이상** (파이프라인은 줄여 쓰지만, 나중 큰 GPU 대비 원본은 고화질로).
- 가로(landscape) 권장. 흔들림 적게(삼각대/거치).
- 길이 **5~11분** 적당.

## 개수
- **3~5개** 찍기 (n=1은 통계 불가). 서로 다른 작업이면 더 좋음.

## 찍은 뒤 할 일
1. 파일을 `data/raw/V04.mp4`, `V05.mp4` … 로 넣기.
2. 오디오 추출: `ffmpeg -i V04.mp4 -ac 1 -ar 16000 V04.wav`
3. **정답지 솔직하게 작성** — "내가 말한 곳"이 아니라 **"실제로 노하우가 있는 곳"**을 표시.
   (조용히 시범한 중요 구간도 정답지에 포함! 말한 것만 적으면 또 순환됨)
4. `conditions.C.enabled=true` 로 켜고 본실험.

## 한 줄 요약
> 전체를 찍되 **말/침묵을 자연스럽게 섞고**, **중요한 걸 말없이 보여주는 구간도 일부러 넣고**, 정답지는 말이 아니라 **실제 노하우 위치**로 매긴다.
