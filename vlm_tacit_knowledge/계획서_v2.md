# 명장 암묵지 추출 — VLM 입력 조건 비교 실험 계획서 (v2, 현행 통합본)

> 2026-06-27 기준. 앞 3연구(STT선정→VAD효과→엔진비교)에 이은 **연구④**.
> 논문 하나의 스토리로 묶음. 파라미터(pad400·turbo·WebRTC) 재사용.

## 1. 가설
> **발화가 있는 곳에 암묵지가 있다.** 명장은 의미 있는 행동을 말로 함께 설명하므로,
> 발화 구간만 봐도 암묵지를 충분히 포착할 수 있다.

## 2. 2단계 구조
- **1단계 (현재) — 방법 검증:** 진짜 영상 + 내가 쓴 정답지로 메커니즘 확인. 명장 검수 불필요.
- **2단계 (후속) — 현실 적용:** 실제 명장 영상 소수로 재현.

## 3. 조건 (입력만 변수, 나머지 전부 고정)
| 조건 | 입력 | 역할 |
|---|---|---|
| **A** | 통영상 균등 32프레임(8장×4청크 병합) + 전체 STT | upper bound 천장 |
| **B** | 발화 구간 프레임 + 구간 STT | 가설 구현 |
| **B_vid** | 발화 구간 프레임만 | ablation: 영상 단독 기여 |
| **B_txt** | 구간 STT만 | ablation: 발화 단독 기여 |
| **C** | B 구간과 동일 길이 랜덤창 + STT | 대조군 (보류) |

- **ablation B vs B_vid vs B_txt** = "영상이 STT 위에 실제 기여하나" 분리 → STT 혼입 교란 방어.
  - B ≈ B_txt & B_vid 낮음 → 출력이 사실상 STT에서 나옴(영상 기여 미미).
  - B > B_txt & B_vid 의미있음 → 영상이 실제 기여.
- **본 실험 메인 = B vs C**(발화 *위치* 효과) — 침묵 충분한 영상 확보 후 활성화.
- 분석 단위 = **구간(segment)** (영상 아님) → 페어 확보, 앞 연구 n 한계 회피.

## 4. 데이터 현황
- **V03** (`data/raw/V03.mp4`): 자전거 타이어 교체 11:16 원본(youtu.be/lTOJkN4vV3Y). 정답지 7구간 일치.
- V01(2분, 폐기)·V02(4분, V03 앞부분).
- **현 한계:** V03 발화 90% → 수동 7구간이 영상 전체 덮음 → **C 대조 불가, B≈A**.
  → 현 단계는 **A vs B(효율 1차) + ablation(영상 기여)** 까지. 가설검증(B vs C)은 침묵 충분 영상 후.

## 5. 평가 (개념 키워드 채점)
- 정답지 `data/ground_truth/V03_answerkey.json`: 수동 7구간 window + `keywords=[[동의어..],..]`.
- 개념 적중 = 동의어 중 하나라도 출력에 등장. 구간 recall = 적중/전체.
- (한계: 동의어 누락=가짜0점, 부분문자열=가짜적중 → 최종은 LLM-judge/사람)

## 6. 하드웨어 / 모델
- 로컬 **Qwen2.5-VL-3B-Instruct 4bit** (RTX2080 8GB 상한, 7B 불가).
- OOM 방지: `max_frames_per_call=8`, `max_pixels=313600`, `expandable_segments`. peak ~2~3GB.

## 7. 진행/완료
- ✅ 전 파이프라인 + Qwen 실추론 동작. ablation·A청크·프레임상한·STT 1회전사 적용.
- ✅ 1차 실측에서 결함 발견→수정: 프롬프트 echo, JSON 잘림, 반복, A 천장붕괴, STT 혼입, 채점 동의어.

## 8. 알려진 한계 (데이터/사람 필요 — 코드 불가)
1. **n=1** → 통계 불가. 영상 다수 필요.
2. **B≈A 구조적** → 발화:비발화 ≈ 50:50 영상 필요(C 성립도 동시 해결).
3. **정답지 순환**(keywords가 STT 기반) → ablation으로 부분 방어, 궁극엔 영상 근거 정답지.
4. **3B/4bit 품질 천장** → 서술 얕음·환각 가능.

## 9. 다음 액션
1. **침묵 충분한 자전거/정비 영상 3~5개** 확보 (50:50). ← 최우선
2. 각 영상 정답지(키워드) 작성 → `make_answerkey_template.py` 후 채우기.
3. **C 활성화**(`conditions.C.enabled=true`) → B vs C 본 실험.
4. 채점 LLM-judge 옵션 검토(동의어/환각 보완).
5. 결과 정리 → 논문 ④절 초안.
