# 연구④ 1차 실험 결과 보고서 — V03 (2026-06-27)

## 1. 무엇을 했나
자전거 타이어 교체 영상(V03, 11분) 1개로, **입력 조건만 바꿔** 로컬 Qwen2.5-VL-3B(8GB·4bit)에 넣고 암묵지 추출 → 정답지(수동 7구간) 개념 키워드로 채점.

## 2. 결과 (파싱 20 OK / 2 복구 / 0 에러 → 신뢰 가능)
| 조건 | 입력 | recall |
|---|---|---|
| **B_txt** | 말(STT)만 | **0.738** |
| **B** | 프레임 + 말 | 0.488 |
| **B_vid** | 영상(프레임)만 | 0.048 |
| A | 전체영상 천장 | 0.273 |

## 3. 알게 된 것
1. 이 영상에선 **노하우가 거의 다 "말"에 있었다** (STT만 0.738).
2. 작은 VLM에 **흐릿한 프레임만 주면 영상에서 거의 못 뽑는다** (0.048), 프레임을 더하면 오히려 떨어진다(B 0.488 < B_txt 0.738).
3. **단, "영상에 정보가 없다"는 뜻이 아니다.** 0.048은 작은 VLM이 흐릿한 8장 안에서 "손이 어디 있고 주걱을 어떻게 쓰는지"를 **스스로 못 찾은** 것 — 정보가 없는 게 아니라 **읽어내질 못한** 것.

## 4. 왜 아직 결론이 아닌가 (한계)
- **n=1**, 게다가 **발화 90% 영상**이라 "말만 들어도 되는" 쉬운 케이스.
- **8GB 제약**으로 8장·448px까지 깎아서 모델 눈을 가렸음 → "영상 무용" vs "모델이 약해 못 읽음" **구분 불가**.

## 5. 수정 방향 — "통프레임" 대신 **행동 정보를 떠먹이기**
흐릿한 그림을 VLM이 헤매게 두지 말고, **YOLO로 손·도구 위치를 좌표로 뽑아 텍스트로** 넣어준다. ("지금 손이 림 위쪽, 주걱을 끝부분에 댐")

**확장 입력 조건:**
| 조건 | 입력 | 의미 |
|---|---|---|
| 말만(STT) | 말 | 베이스라인 0.738 |
| 프레임+말 | 영상+말 | 현재 0.488 (오히려 ↓) |
| **YOLO+말** | 손·도구 위치 + 말 | **새 후보** |
| YOLO+프레임+말 | 다 합침 | 새 후보 |

> **핵심 질문: YOLO 행동 정보를 넣으면 말만(0.738)을 넘는가?**
> 넘으면 = "영상(행동)이 진짜 기여한다" → 가설의 핵심 증거.

**분석 신호 3종 (행동 정보 추출):**
| 신호 | 잡는 법 | 의미 |
|---|---|---|
| 손·도구 | YOLO 객체탐지 | 무슨 작업 중 |
| 주의 전환 | YOLO로 화면 중앙 객체 추적 | 점검/판단 우선순위 |
| 위치 이동 | optical flow / 장면 전환 | 단계 전환 |

## 6. 환경 제약 (왜 이 고생인가)
회사 **내부 폐쇄망/온프렘**이라 데이터를 밖으로 못 보냄 → **Claude 같은 외부 VLM API 사용 불가**(API 전용, 로컬 가중치 없음). → **로컬 강제** → 8GB GPU(RTX2080) → 소형 Qwen 3B·저화질·소수 프레임 → 영상을 못 읽음. **큰 GPU/클라우드면 안 겪을 문제.** (그래서 "행동 정보를 미리 뽑아 떠먹이는" 우회가 합리적.)

## 7. 평가 / 권고
- **YOLO+STT 방향 = 옳음.** 3번 진단("못 읽은 것")에 대한 정확한 처방. 통프레임을 키우는 것(8GB 한계)보다, 행동을 구조화해 떠먹이는 게 폐쇄망·소형모델 제약에 맞는 정공법.
- 단 이걸 주장하려면 **데이터가 받쳐줘야** 함: 발화/침묵 섞인 영상(촬영가이드 참조) + YOLO·STT가 제대로 잡히는 작업.

## 8. 데이터셋이 다 갖춰지면 (실행 계획)
1. 시나리오 촬영(발화:침묵≈5:5, 4칸 섞기) 영상 3~5개 + 정답지(실제 노하우 위치 기준).
2. 전처리: VAD·STT(말) + **YOLO(손·도구)** + optical flow/장면전환(이동) → 구간별 행동 텍스트.
3. 입력 조건 4종(말만 / 프레임+말 / YOLO+말 / 전부) × 구간 단위 + **C(랜덤창)** 켜기.
4. 채점: 개념 키워드 + (가능하면) LLM-judge 보완. B vs C·말만 대비 Wilcoxon.
5. 결론: "행동 정보가 말 위에 기여하나"로 가설 판정 → 논문 ④절.

## 9. 바로 다음 (둘 중)
- (a) **빠른 확인**: 화질만 올려(8GB 스윕) 영상 점수가 움직이나 본다.
- (b) **정식**: 시나리오 촬영 + YOLO·STT 정비 → 위 8번 실행.
