# 음성·영상 AI 파이프라인 연구 종합 보고서

> 폐쇄망·온프렘(RTX 2080 8GB) 제약 하에서 한국어 STT부터 멀티모달 암묵지 추출까지,
> **단일 변수 통제 + 통계 검정 + 정직한 한계 명시**를 일관 원칙으로 진행한 4개 연구.
> 기간: 2026-06-24 ~ 06-27 | STT 백엔드: faster-whisper large-v3-turbo (1번에서 선정 후 전 연구 고정)

연구는 로드맵으로 이어진다:
**① 어떤 STT를 쓸까 → ② STT에 VAD를 붙일까 → ③ 붙인다면 어떤 VAD → ④ 영상까지 확장(VLM 암묵지)**

## 1. STT 모델 선정 (`stt_comparison_research/`)

**목적** — 한국어 롱폼에서 STT 3종(large-v3 / large-v3-turbo / medium-ko)을 비교해 배포 모델 1개 선정.
**데이터** — 5파일(낭독 4 + 대화 1 레거시), 16~73분.
**평가지표** — `CER`(문자오류율, 주지표) · `RTF`(실시간 배율, 낮을수록 빠름) · `del_rate`(누락률).
**선정 룰** — *RTF ≤ 0.10 통과 후 CER 최소* (배포 가능 속도를 만족한 것 중 가장 정확한 모델).

**가장 중요한 표 — 선정 결과**

| file | 선정 모델 | CER | RTF |
|---|---|---|---|
| F02 | turbo | 0.044 | 0.028 |
| F03 | turbo | 0.075 | 0.027 |
| F05(롱폼) | turbo | 0.125 | 0.038 |
| F04 | large-v3 | 0.113 | 0.071 |
| xsbdRlpLYhc | turbo | 0.116 | 0.048 |

→ **turbo 4 / large-v3 1 → 최종 `large-v3-turbo` 선정** (CER 평균 turbo 0.108 vs large-v3 0.127 vs medium-ko 0.269).

**유의할 점**
- **Wilcoxon n=5는 최소 p=0.0625 → p<0.05 도달 구조적 불가.** 그래서 선정은 *통계*가 아니라 *룰* 기반, Wilcoxon은 방향성 보조로만.
- medium-ko는 F04에서 **CER 75%·del_rate 73%로 붕괴**(발화를 침묵으로 오판) — 한국어 파인튜닝이 일반화를 해친 사례.
- 시각화 4종: `STT모델비교-시각화/` (CER-RTF 산점도, 롱폼 드리프트 등).

## 2. VAD 전처리 효과 (`vad_stt_research/`, Phase 1)

**목적** — "turbo에 VAD를 붙이면 빨라지고 정확해지나?" (VAD는 Silero 고정, **전처리 방식만** 비교).
**데이터** — 10파일(high_silence 5 + low_silence 5), 53~112분.
**평가지표** — `WER`·`CER`(정확도) · `할루시네이션/h`(없는 말 생성) · `RTF`(속도).
**핵심 조건 (3-arm)** — `A`(vanilla) / `A′`(파라미터 `condition_on_previous_text=False`만) / `B`(VAD+파라미터). 비교축: A→A′=파라미터 효과, A′→B=VAD 순효과.

**가장 중요한 표 — 3축 Wilcoxon (n=10)**

| 지표 | A→A′ (파라미터) | A′→B (VAD 추가) |
|---|---|---|
| WER | 0.299→0.225 ✔개선(p=0.020) | 0.225→0.266 악화 |
| CER | 0.172→0.094 ✔개선(p=0.002) | 0.094→0.133 악화 |
| 할루시네이션/h | 272→100 ✔개선(p=0.002) | 100→119 ✔**악화**(p=0.027) |
| RTF | 0.034→0.021 ✔개선(p=0.002) | 0.021→0.038 ✔**느려짐**(p=0.002) |

→ **A′ 채택, VAD(B) 비권장.** 개선의 실체는 **파라미터 한 줄**이고, VAD는 그 위에서 오히려 속도·할루시네이션을 악화.

**유의할 점**
- 속도 **손익분기 없음** — turbo의 A′ RTF가 이미 0.02대라 Silero 비용(RTF≈0.03)을 무음 제거로 회수 불가.
- 가설3(padding) **지지**: `speech_pad_ms` 0→400으로 WER 단조 감소 → VAD 쓸 거면 **pad ≥ 400** 권장.
- 후반 타임스탬프 드리프트는 세그먼트 수 교란+표본부족(n=6)으로 **결론에서 제외(한계로 강등)** — 측정 신뢰도를 정직하게 인정.
- 시각화 5종: `results/figures/` (정확도-할루시, 속도 워터폴, 손익분기, 민감도, 드리프트).

## 3. VAD 엔진 비교 (`vad_model_comparison/`)

**목적** — "그래도 VAD를 쓴다면 Silero vs WebRTC 중 무엇?" (단일 변수=VAD 엔진만 변동, 나머지 Phase 1 재사용).
**데이터** — Phase 1과 동일 10파일.
**평가지표** — `VAD RTF`(실행비용) · 검출 `F1/Precision/Recall` · downstream `WER/CER/할루시`.
**핵심 조건** — `참조`(VAD없음=A′) / `Silero` / `WebRTC`.

**가장 중요한 표 — 비용·검출·downstream**

| | VAD RTF | 검출 F1 | WER | 할루시/h |
|---|---|---|---|---|
| 참조(VAD없음) | — | — | **0.224** | **100** |
| Silero | 0.0270 | 0.596 | 0.266 | 119 |
| WebRTC | **0.00023** | **0.676** | 0.278 | 128 |

→ **WebRTC가 116배 빠르고(p=0.002) F1도 우위(p=0.004).** 하지만 **두 엔진 다 "참조(VAD없음)"를 못 이김** — WER·CER·할루시 모두 참조가 최저.

**유의할 점**
- **결론: "쓴다면 WebRTC, 근본은 VAD 비권장"** (Phase 1 재확인). 엔진 선택은 그 다음 문제.
- **검출 F1 ≠ downstream WER**: WebRTC가 F1 높아도 WER은 더 나쁨(과검출이 청크를 잘게 쪼개 문맥 단절).
- F1의 GT가 수동자막이라 **recall 높은 WebRTC에 유리하게 편향** 가능 — "본질적 우위"로 과대해석 금지.
- 시각화 4종: `results/figures/` (cost, detection, wer, tradeoff).

## 4. VLM 영상 암묵지 추출 (`vlm_tacit_knowledge/`) — 진행 중(파일럿)

**목적** — 작업 영상에서 VLM(Qwen2.5-VL-3B, 로컬)으로 **암묵지**(말로 안 드러나는 요령)를 추출하고, **입력 구성**을 비교.
**가설** — "발화가 있는 곳에 암묵지가 있다."
**데이터** — 자전거 타이어 교체 11분 1개(파일럿), 정답지 7구간 수동 작성.
**평가지표** — `개념 키워드 recall`(정답지의 핵심 개념이 출력에 등장한 비율).
**핵심 조건 (ablation)** — `B_txt`(말만) / `B`(프레임+말) / `B_vid`(영상만) — 영상의 순기여를 분리.

**가장 중요한 표 — 입력별 recall**

| 조건 | 입력 | recall |
|---|---|---|
| B_txt | 말(STT)만 | **0.738** |
| B | 프레임+말 | 0.488 |
| B_vid | 영상만 | 0.048 |

→ 이 세팅에선 **노하우가 거의 다 "말"에 있었고, 흐릿한 프레임은 오히려 방해**(B < B_txt).

**유의할 점 (★결론 아님)**
- **B_vid=0.048은 "영상에 정보 없음"이 아니라 "8GB 제약으로 8장·저화질로 깎아 모델이 못 읽은 것".** 단독 해석 금지.
- **n=1 + 발화 90% 영상**이라 통계 불가·태생적 쉬운 케이스 → **방법 검증용 파일럿**으로만 유효.
- **다음 처방**: 통프레임 대신 **YOLO로 손·도구 위치를 좌표 텍스트로 떠먹이기** + 발화/침묵 5:5 시나리오 영상. 핵심질문은 "행동정보가 말(0.738)을 넘나".
- 1차 디버깅 성과: 출력 19/22 파손(반복옵션이 JSON 키 변형·언어 드리프트)을 원인 분리·수정해 **에러 0**까지 복구.

## 전체를 관통하는 발견·방법론

1. **"좋은 전처리"라는 통념을 데이터로 뒤집음** — VAD는 turbo 환경에서 도움이 안 됨(2·3). 직관이 아니라 측정으로 결론.
2. **일관된 실험 위생** — 모든 연구가 *단일 변수 통제 → Wilcoxon → 한계 명시* 구조. 안 되는 것(n=5 유의 한계, 드리프트 측정 한계, VLM n=1)을 **숨기지 않고 명기**.
3. **제약을 정면으로 다룸** — 폐쇄망(외부 API 불가)·8GB GPU라는 현실 위에서 모델 선정·메모리 최적화·우회 설계까지 수행.
4. **로드맵의 자연스러운 확장** — 음성(1~3)에서 멀티모달(4)로, 각 단계가 다음 단계의 입력·전제를 만든다.

> **현재 위치:** 1~3 완료, 4는 파이프라인 검증 완료·본실험(시나리오 영상) 대기.
> **다음 병목:** 모델·화질이 아니라 **데이터(발화/침묵 섞인 시나리오 영상)**.
